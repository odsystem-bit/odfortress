# Contribuer à OD Fortress

Merci de votre intérêt pour OD Fortress ! Ce document présente les lignes directrices pour contribuer au projet.

> **Note** : OD Fortress est actuellement en développement actif. Le code source applicatif reste privé pour le moment. Ce dépôt public est dédié à la documentation, à la présentation du projet et aux échanges avec la communauté.

## Comment contribuer

### 1. Signaler un problème

Si vous détectez une erreur dans la documentation, une incohérence ou une suggestion d'amélioration :

- Ouvrez une **issue** sur GitHub.
- Décrivez le problème avec précision.
- Proposez une solution si vous en avez une.

### 2. Proposer une modification de la documentation

- Forkez le dépôt.
- Créez une branche dédiée : `docs/nom-de-la-modification`.
- Apportez vos modifications.
- Soumettez une **pull request** claire et concise.

### 3. Discuter de la vision et du produit

Les discussions sur l'architecture, les fonctionnalités et le modèle économique sont les bienvenues dans les **GitHub Discussions**.

## Conventions de rédaction

- Utilisez un français clair et professionnel.
- Structurez les documents avec des titres, des listes et des tableaux.
- Préférez les placeholders (`À compléter`) plutôt que des informations approximatives.
- Citez les sources et les références lorsque c'est pertinent.

## Code de conduite

Tous les contributeurs doivent respecter le [`CODE_OF_CONDUCT.md`](CODE_OF_CONDUCT.md).

## Questions

Pour toute question, contactez l'équipe via les coordonnées disponibles dans le [`README.md`](README.md).
